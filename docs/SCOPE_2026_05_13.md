## What is done

### 1. Foundation — Phase 1 (gate G2 passed)

Everything a teen needs to land in the product and be safe:

- **Auth & parental consent.** Clerk-backed teen sign-up; under-18 flow captures parent email and gates progression on a Resend-delivered consent link (`workers/src/lib/consent.ts`, `users.parent_consent_at` / `parent_email`). Consent action is logged in D1 and audit-able.
- **Onboarding & engine routing.** Six-question intake → LLM-written 3-sentence summary → structured engine pick → frontend lands the teen in the right engine. Falls back to a deterministic keyword router when AI is unavailable. (`workers/src/routes/user.ts:77`, `workers/src/lib/intake.ts`)
- **Kai customization.** Teen picks Kai's name (default "Kai") and tone (warm / balanced / direct). Persisted on `users.kai_name` / `users.kai_tone` and substituted into every Kai prompt.
- **Safety classifier (live in front of every model call).** Two-layer:
  - **Regex fast-path** for exact-phrase crisis language. Catches the unambiguous cases in microseconds. (`workers/src/lib/safety.ts:47`)
  - **Structured-output LLM** (currently `@cf/meta/llama-3.1-8b-instruct`; same prompt swaps to Claude in one config change). Catches paraphrased crisis language the regex misses. (`workers/src/lib/safety.ts:127`)
  - **27 adversarial paraphrases** across all six categories — suicide ideation, self-harm, eating disorder, abuse disclosure, substance, violence to others — covered by automated tests so future prompt changes can't silently regress. (PR #68, in CI.)
- **Crisis page (always-on).** Static `/crisis` route, footer link on every page, PWA service worker pre-caches it offline. Region-aware: US/Canada (988), UK/Ireland (Samaritans/Childline/Shout), Australia/New Zealand (Lifeline/Kids Helpline), international (IASP directory). (`src/pages/Crisis.tsx`, `src/lib/crisis-resources.ts`, `public/sw.js`)
- **Parent notification for critical events.** When the classifier flags a critical-severity message in suicide/self-harm/abuse categories, the parent on file is emailed with a calm, content-free summary (no raw teen text). 24-hour cooldown per user × category. (`workers/src/lib/safety.ts:245`, `sendParentSafetyAlert` in `email.ts`)
- **Operations review surface.** `/api/ops/safety-events` lists every safety event for org-admin Clerk users. A frontend `/ops` page renders the list sorted by severity. Daily ops review is now possible. (`workers/src/routes/ops.ts`, `src/pages/Ops.tsx`)
- **Terms / Privacy.** Placeholder copy in place at `/terms` and `/privacy` — flagged for legal review (D6).

### 3. Cross-cutting product

- **Progress tracker.** Server-computed level (Section 9 ladder: 0/100/250/500/1000/1750/2750/4000/6000/9000), per-engine and overall streaks, belt milestones. KV-cached for cheap dashboard hits, invalidated on event insert. (`workers/src/lib/levels.ts`, `workers/src/lib/progress.ts`)
- **Nightly recompute cron.** Cloudflare scheduled trigger at 07:00 UTC walks every user and refreshes their summary so streaks decay correctly for inactive users — without it, a teen who skipped three days would keep an inflated streak. (`wrangler.worker.toml`, `workers/src/index.ts` `scheduled` handler)
- **Evolving character.** Procedural SVG that visibly changes across 10 levels (size, eye brightness, mouth curve, aura, motion). Real commissioned art is on hold for D1. Works on Day 1 without art assets. (`src/components/tracker/EvolvingCharacter.tsx`, `src/lib/avatar.ts`)
- **Friend compare.** Opt-in, aggregate-only. Shows streaks, level, weekly score across mutual friends. Never shows conversation content, goal text, meal photos, or reflection content. (`src/components/tracker/FriendCompare.tsx`, `workers/src/routes/friends.ts`)
- **`prefers-reduced-motion` honored everywhere.** Tailwind `motion-safe:` on explicit animations, plus a belt-and-suspenders global rule in `globals.css` that nullifies every transition and animation when the OS preference is set. Verified across all 3 engines.

## Risks and unknowns to flag

Listed bluntly. Not exhaustive.

1. **The classifier is good but not perfect.** Llama-3.1-8b is the current safety brain. It's better than regex alone (PR #68 documents the paraphrases the regex misses), but it will occasionally miss things and occasionally false-positive. The Anthropic swap (D3) reduces both error rates; the operations review (daily) catches what slipped through. Real-teen rollout should be small and observed.

2. **No real teen has used this yet.** Every flow is internally tested. We have no signal on whether teens will *actually* pick up the food-photo logger, run the breath recommender, write a future-self letter. Phase 5 friend-test is when we learn that.

3. **Parental notification is a hard promise.** When critical safety lands, the parent gets emailed. The cooldown is 24h per user × category — that's tuned to balance "don't spam" against "don't miss a second event." Worth a stress-test with a clinician once D5 lands.

4. **Llama uses Workers AI (free tier today).** When usage grows, costs apply. Hard to project until real teens are on the platform. Worth a back-of-envelope before launch — `n teens × m messages/day × k tokens × $/token`.

5. **The mobile-first design rebuild (after D1) is meaningful work.** "Design parked" means the current UI is the May 11 baseline plus the in-flight mobile-first refinements (#63, #64). Once Lev picks, expect 2–3 weeks of engine UI work.

6. **Resend domain status.** `hello@kai.boostaisearch.ai` is verified. If we ever change the sending domain, parent consent emails go to spam until re-verified.

7. **No PII review yet.** D1 stores teen email, parent email, intake responses. We redact safety event text but not intake answers. Worth a privacy audit (consider it part of D6 legal review).
